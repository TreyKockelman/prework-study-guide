HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • "title" defines the title for the webpage and is extremely important for Search Engine Optimization.
  • "styles" defines CSS styles associated with the webpage.
  • "base" defines base URL for a webpage.
  • "link" connects external resources to HTML Doc.
  • "meta" defines metadata: character sets, descriptions, keywords, author, and viewpoints.
  • "script" embeds client-side scripts in HTML Doc.

CSS

  • CSS defines colors, aesthetic, layout, and organization.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Most preferred method of CSS is external CSS style sheets.
  • Class attributes allow us to share a CSS Rule to any element we choose.
  • CSS is linked in the head of the HTML file.

GIT

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add: adds modifications to the staging area
  • -A: indicates we want to add all changes
  • git commit: creates commit that captures screenshot of currently staged changes
  • git pull: receives branch modifications into local environment
  • Always do a pull to make sure you are on the latest version of a project on Local
  • git push: pushes changes to remote Git branch

JavaScript

  • JavaScript adds interactivity to a webpage
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript is linked in the body of HTML file.
  • A function is a set of instructions that tells the computer how to perform a certain task.
  • A function has to be called by name to execute.
